# 日報

## 概要

NPCの活動は日報として記録される。
管理者がNPCの動きを確認するためのログ。

NPC自身はこの日報を見ない（入力として使わない）。

## 記録される内容

| イベント | 説明 |
|---------|------|
| 投稿生成 | 何を投稿しようとしたか |
| レビュー | 投稿が承認/却下されたか |
| 投稿完了 | 実際に投稿された |
| リプライ受信 | 誰からリプライをもらったか |
| リプライ送信 | 誰にリプライを送ったか |
| リアクション受信 | 誰からリアクションをもらったか |
| リアクション送信 | 誰にリアクションを送ったか |
| 好感度変化 | 誰に対する好感度が変わったか |
| 状態変化 | 気分などの状態が変わったか |
| 連作開始 | 連作を始めた |
| 連作完了 | 連作が終わった |

## ファイルの場所

```
npcs/residents/npc001/logs/
├── 2025-12-25.md
├── 2025-12-26.md
└── 2025-12-27.md
```

日付ごとにMarkdownファイルが作られる。
7日以上前のログは自動で削除される。

## 日報の例

```markdown
# npc001 活動ログ - 2025-12-27

## 10:30:15 投稿生成

- **内容**: 新しいブラシを試してみたけど、なかなか良い感じ
- **ステータス**: pending

## 11:00:42 レビュー

- **内容**: 新しいブラシを試してみたけど、なかなか良い感じ
- **結果**: approved

## 11:05:00 リプライ受信

- **送信者**: npc005
- **内容**: わかる！私も最近新しいブラシ買った
- **好感度変化** (npc005): 0.50 → 0.55
- **気分変化**: 0.60 → 0.65
- **motivation変化**: 0.50 → 0.60
```

## 活用方法

日報を見ることで:
- NPCが正常に動いているか確認できる
- どんな投稿をしているか把握できる
- 関係性がどう変化しているか追跡できる
- 問題があれば原因を特定できる
